## Overview

This project is a **Pharmacy Management System (Front End only)** designed to provide a clean and simple user interface for managing pharmacy-related data such as medicines, categories, and basic operations.

The system is currently implemented as a **front-end application only**.  
A backend service will be developed and integrated later by a backend developer.

---

## Project Type

- Front End: Yes  
- Back End: Not included (to be implemented separately)

---

## Tech Stack

- React
- TypeScript
- Vite
- Tailwind CSS
- Lucide Icons

---

## Features

- Pharmacy-oriented user interface
- Medicine listing and categorization
- Clean and responsive layout
- Typed data models using TypeScript
- Ready for backend integration

---

## Color Palette

The project uses a simple and clean color system focused on clarity and usability.

- Green  
  Used for primary actions, confirmations, buttons, and highlights related to success and availability.

- Dark Gray / Black  
  Used for main text, headers, and important labels.

- Light Gray  
  Used for backgrounds, cards, borders, and layout separation.

- White  
  Used for main backgrounds and content containers.

- Red  
  Used for warnings, delete actions, and critical alerts.

## Project Structure

- `App.tsx`  
  Main application component.

- `index.tsx`  
  Application entry point.

- `constants.tsx`  
  Static data and predefined values.

- `types.ts`  
  TypeScript interfaces and data models.

- `vite.config.ts`  
  Vite configuration.

- `index.html`  
  HTML template.

---

## Notes

- This project does not include authentication or database connectivity.
- All data is currently handled on the front end.
- The structure is prepared for easy backend integration in the future.
